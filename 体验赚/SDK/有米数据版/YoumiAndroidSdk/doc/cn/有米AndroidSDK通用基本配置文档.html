    有米 Android SDK 通用基本配置开发者文档¶

    小技巧

    欢迎您使用有米广告 SDK，为了保证能正常接收有米的广告，保障各位开发者的收益，请您严格按照 SDK 里 doc 文件夹下的文档步骤来嵌入 SDK ，或按照 SDK 里的 demo 源码来嵌入。

    以下注意事项，请仔细阅读：

    1. 在有米后台创建应用获取发布 ID 和应用密钥时，请注意区分好 Android 和 iOS 平台，不能弄错平台。
    2. 一对 ID 和应用密钥只能用在一个应用内，不能重复用在多个应用内，否则会拿不到正式广告。
    3. 嵌入完 SDK，可以拿到测试广告，测试没问题，需上传应用给有米审核，审核通过后才能拿到正式广告，正式广告才能产生收益。
    4. 有米广告不支持模拟器，测试广告时请用真机来测试，另外，Android 广告不支持 Android 平板，需要用手机来测试 Android 广告。
    5. 应用 提交市场审核前 请注意：不要在界面或代码中出现类似 ym, youmi, ad 等与有米或者广告相关的字眼；记得隐藏广告以及广告入口；打包应用时开启混淆相关配置。

    一、基本配置¶

    1、导入 SDK¶

    将 sdk 解压后的 libs 目录下的 YoumiSdk_*.aar 文件导入到工程指定的 libs 目录

    小技巧

    PS :

    你可能注意到sdk解压后还存在一个 extralibs 目录，该目录存放我们和其他第三方厂商合作所需要用到的扩展jar，属于 可选导入的jar ，只有你用到某些功能的时候，才需要导入，在你使用该功能的时候，该功能的文档会说明需要引入哪一个jar

    e.g :

    • miadtracker.jar : 小米游戏中心渠道统计sdk，当你的 应用/游戏需要发布到小米游戏中心 的时候，还需要导入本jar到你的工程项目的 libs 目录中

    2、添加依赖¶

    在应用的 build.gradle 中添加以下配置：

    如果你使用 Android Gradle Plugin 3.0.0 以上版本，建议使用下面的方法引入 AAR：

    dependencies {
    implementation fileTree(dir: 'libs', include: ['*.aar'])
}

    如果你使用 Android Gradle Plugin 3.0.0 以下版本，请使用下面的方法引入 AAR：

    repositories {
    flatDir {
        dirs 'libs'
    }
}

dependencies {
    // name 为上一步导入 SDK 中导入到工程的 AAR 文件名去除文件后缀的字符串。
    // 比如上一步导入 SDK 中导入到工程的 AAR 文件名为 YoumiSdk_v7.2.1_2017-05-04.aar，
    // 那么这里应该这样配置 compile name: 'YoumiSdk_v7.2.1_2017-05-04', ext: 'aar'
    compile(name: 'YoumiSdk_*', ext: 'aar')
}

    3、指定支持的so架构¶

    在应用的 build.gradle 中指定应用支持的so架构类型，开发者根据自身产品实际情况，酌情处理

    如：下方表示只支持 armeabi 架构

    android {
    defaultConfig {
        // 指定支持的so架构
        ndk {
            abiFilters 'armeabi'//, 'armeabi-v7a', 'arm64-v8a', 'x86', 'x86_64', 'mips', 'mips64'
        }
    }
}

    4、编译打包¶

    在应用的 build.gradle 中指定编译SDK版本为 24 以上（包括24）

    android {
    compileSdkVersion 24
}

    5、初始化应用信息¶

    请务必在应用第一个 Activity（启动的第一个类）的 onCreate 中调用以下代码

    import cde.ewd.adw.AdManager;
...
AdManager.getInstance(Context context).init(String appId, String appSecret, boolean isEnableYoumiLog);

    小技巧

    参数说明：

    • appId 和 appSecret 分别为应用的发布 ID 和密钥，由有米后台自动生成，通过在有米后台 > 应用详细信息 可以获得。
    • isEnableYoumiLog: 是否开启有米的Log输出，默认为开启状态
      • 上传到有米主站进行审核时，务必开启有米的Log，这样才能保证通过审核
      • 审核通过后如需请求测试广告，请到 有米后台-应用中心-广告设置-测试设备设置 进行设置
      • 开发者发布apk到各大市场的时候，强烈建议关闭有米的Log

    6、设置渠道号（可选）¶

    目前，有米已经与多家应用商店合作，当您发布的目标应用商店有要求您加上有米的渠道号时，您可以按以下操作加入渠道号进行打包，这样可以保证您的应用在该应用商店上顺利审核通过。

    渠道号列表详见 有米广告推广渠道号列表

    在AndroidManifest.xml中添加以下代码:

    <meta-data
    android:name="YOUMI_CHANNEL"
    android:value="这里替换为非负整数的渠道号" >
</meta-data>

    6.1 小米游戏商店¶

    有米跟小米游戏商店达成合作，如果你的应用/游戏需要上传发布到小米游戏商店，那么只需要做额外一点工作，就可以保证您的应用能顺利通过小米游戏商店的审核。

    1. 在 AndroidManifest.xml中填写小米游戏中心的渠道号 : 11030
    2. 在你的项目中 libs 目录下导入添加我们sdk解压后目录中的小米游戏商店的统计sdk(extralibs/miadtracker.jar)
    3. 最后，加入该sdk的混淆配置：
    -keep public class com.mi.adtracker.MiAdTracker{ *; }

    小技巧

    至此，恭喜您已经准备好有米Android SDK的通用基本配置。
